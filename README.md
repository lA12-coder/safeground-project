**SafeGround** is a comprehensive recovery support platform built specifically for **Ethiopian university students** struggling with khat addiction. Combining the power of AI-driven support, community connection, and faith-based encouragement, SafeGround provides a holistic, private, and dignified path to recovery.

### 💔 The Problem

Khat (ጫት) addiction affects a significant portion of Ethiopian university students, often leading to:
- 📉 **Academic decline** and dropped courses
- 💰 **Financial strain** on students and families
- 🧠 **Mental health challenges** (anxiety, depression, isolation)
- 🤝 **Strained relationships** with family and peers
- 🕊️ **Spiritual disconnection** from faith communities

### 💡 Our Solution

SafeGround addresses these challenges through a **privacy-first, multi-stakeholder platform** that brings together:

| Stakeholder | What They Get |
|---|---|
| 🎓 **Students** | Anonymous tracking, AI companion, crisis support, spiritual guidance |
| 👨‍👩‍👧 **Guardians** | Read-only support view, encouragement tools, milestone notifications |
| 👨‍⚕️ **Providers** | Verification, booking management, encrypted session notes |
| 🏛️ **Organizations** | Wellness portal, program management, impact metrics |
| 🛡️ **Admins** | Real-time metrics, moderation, seed data, system health |

### 🔒 Security Architecture

```
┌──────────────────────────────────────────────────────────────┐
│                    SECURITY LAYERS                           │
├──────────────────────────────────────────────────────────────┤
│  🛡️  Middleware: Route-level access control                 │
│  🔐  Supabase RLS: Database row-level security               │
│  👤  Anonymous Aliases: No PII in UI                         │
│  🔑  Guardian Tokens: 32-byte cryptographic tokens           │
│  🔒  Service Role Key: Server-only admin operations          │
│  🚫  Soft Deletes: Preserves data integrity for moderation   │
│  🧹  Rate Limiting: In-memory guest chat limits              │
└──────────────────────────────────────────────────────────────┘
```

### 🗺️ Route Map (Middleware)

| Path | Access | Notes |
|---|---|---|
| `/`, `/guest`, `/register`, `/login`, `/onboarding` | 🌍 Public | No authentication |
| `/guardian/*`, `/org/register` | 🌍 Public | Token-gated |
| `/admin/*` | 🔐 Admin | Email in `ADMIN_EMAILS` |
| `/dashboard/*`, `/settings/*`, `/provider/*`, `/org/portal/*` | 🔐 User | Auth + onboarding done |

## 🗄️ Database Setup

### Step 1: Create a Supabase Project

1. Go to [supabase.com](https://supabase.com) and create a free account
2. Click **"New Project"**
3. Choose a name, database password, and region closest to Ethiopia (e.g., **Frankfurt** or **Singapore**)
4. Wait ~2 minutes for provisioning

### Step 2: Run the Schema

1. In your Supabase dashboard, go to **SQL Editor** (left sidebar)
2. Click **"New Query"**
3. Copy the entire contents of [`scripts/schema.sql`](scripts/schema.sql)
4. Paste and click **"Run"** (or press `Ctrl/Cmd + Enter`)

This will create all required tables:
- `profiles` — User profiles with aliases
- `streaks` — Recovery streak tracking
- `habit_logs` — Daily check-in history
- `panic_events` — Crisis intervention logs
- `guardians` / `guardian_controls` — Guardian relationships
- `providers` — Healthcare/spiritual/community providers
- `telehealth_bookings` — Session scheduling
- `anonymous_chat` — Community chat messages
- `organizations` — Partner organizations
- `faith_programs` — Spiritual program enrollments

### Step 3: Configure Authentication

In Supabase Dashboard → **Authentication** → **Providers**:
- ✅ Enable **Email** (default)
- 🔧 Optional: Enable **Google**, **Phone (OTP)**, etc.
- 🔧 Configure email templates to match SafeGround branding
